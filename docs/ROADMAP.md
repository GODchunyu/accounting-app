# 记账 APP 开发路线图

版本：v1.0  
日期：2026-07-05  
关联文档：[PRD.md](PRD.md)  
策略：先跑通个人记账核心闭环，再做高保真视觉和后期 App 化准备。

## 1. 技术栈

| 层 | 选型 | 说明 |
|----|------|------|
| 前端 | React + Vite + TypeScript | 移动端 H5 高保真原型 |
| 前端样式 | 自定义 CSS + lucide-react | 避免大型 UI 库影响高保真复刻 |
| 后端 | Node.js + Express + TypeScript | 轻量 REST API |
| ORM / 迁移 | Prisma | 数据模型、迁移、类型生成 |
| 数据库 | PostgreSQL | 主数据存储，金额使用 numeric / Decimal |
| 认证 | JWT + bcrypt | 用户名密码登录 |
| 图片 | 本地 uploads + Docker volume | 第一版图片凭证持久化 |
| 部署 | Docker Compose | 管理 API 与数据库 |

## 2. 核心闭环

```text
注册登录 -> 自动创建默认账本和分类 -> 记一笔 -> 明细查看 -> 图表统计 -> 管理分类/账本
```

所有任务优先服务于该闭环。

## 6. 依赖关系

```text
Phase 0 -> Phase 1 -> Phase 2 -> Phase 3 -> Phase 4
             |          |          |
             |          |          -> 高保真依赖真实页面和统计 API
             |          -> 前端依赖后端核心 API
             -> 后端依赖数据库和 Prisma schema
```

关键依赖：

- `R-010` 是所有后端业务的基础。
- `R-011` 和 `R-012` 是用户数据隔离与默认数据的基础。
- `R-013`、`R-014` 是 `R-015` 账单 CRUD 的前置。
- `R-016` 图片上传依赖账单创建/编辑流程。
- `R-017`、`R-018` 统计接口依赖账单数据。
- 前端页面必须接真实 API，不长期保留假数据。
